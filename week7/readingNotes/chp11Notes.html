Chp 11 Notes: Further Functions

Errors, Exceptions, and Warnings

  • Cause by one of the following:
    • System error ― there’s a problem with the system or external devices with which the program is interacting.
    • Programmer error ― the program contains incorrect syntax or faulty logic; it could even be as simple as a typo.
    • User error ― the user has entered data incorrectly, which the program is unable to handle.
  • Exception
    • Error that produces return value
    • Stack Trace: works backwards from the point where the error occurred to identify the original function or method that started the sequence
    • Warnings: there isn’t an error in code big enough to cause program to crash. Program still runs after warning.

Importance

  • In development = code fails should be loud making errors easy to identify
  • In production = code fails should be graceful but not silent so it doesn’t affect user experience

Strict mode

  • Add ‘use strict’; to first line of JavaScript file which will produce more exceptions and warnings. Poor style now becomes errors.

Linting Tools

  • JS Lint, JS Hint, ES Lint
  • They highlight any sloppy programming practices or syntax errors.

Throwing Exceptions

  • Use throw statement to throw your own exceptions
  • Best to throw an error object instead of a integer or string
    • Example: if the number is less than 0 which would return a NaN for Math.sqrt() then you can do this:
      • function squareRoot(number) {
      • 'use strict';
      • if (number < 0) {
      • throw new RangeError('You can't find the square root of negative numbers')
      • }
      • return Math.sqrt(number);
      • };

 

Exception Handling

  • There are methods to identify an error and still let the user have a normal experience instead of crashing the program
  • Try: if we think some code will result in an exception then wrap in a try block. This runs the code inside the block as normal, but if exception occurs it will pass the error object onto a catch block.
    • Example: Code inside the catch block will only run if an exception is thrown inside the try block. The error object is automatically passed as a parameter to the catch block.
      • function imaginarySquareRoot(number) {
      • 'use strict';
      • try {
      • return String(squareRoot(number));
      • } catch(error) {
      • return squareRoot(-number)+'i';
      • }
      • }
    • Finally: can be added after a catch block. This is always executed after the try or catch block regardless if an exception happened or not. Do this if you want some code to run in both cases.
      • Example:
        • function imaginarySquareRoot(number) {
        • 'use strict';
        • let answer;
        • try {
        • answer = String(squareRoot(number));
        • } catch(error) {
        • answer = squareRoot(-number)+"i";
        • } finally {
        • return `+ or - ${answer}`;
        • }
        • }

Function Properties

    • function square(x) {
    • return x*x;
    • }
    • length
    • << 1

Call and Apply

  • Can be used to pull information out of an object/array
  • Example
    • This function exists
      • function sayHello(){
      • return `Hello, my name is ${ this.name }`;
      • }
    • Create objects with the name property then use call() to invoke say() and provide an argument
      • const clark = { name: 'Clark' };
      • const bruce = { name: 'Bruce' };
      • call(clark);
      • << 'Hello, my name is Clarke'
      • call(bruce);
      • << 'Hello, my name is Bruce'
    • If the function being invoked requires any parameters then that needs to be provided after the first argument
      • function sayHello(greeting='Hello'){
      • return `${ greeting }, my name is ${ this.name }`;
      • }
      • call(clark, 'How do you do');
      • << 'How do you do, my name is Clark'
      • call(bruce);
      • << 'Hello, my name is Bruce'

Immediately Invoked Function Expressions (IIFE – “iffy”)

  • These functions run as soon as they are defined. Parentheses are placed at the end of the function.
  • This keeps variables contained to functions and keeps the global namespace clean
  • Example
    • (function(){
    • const temp = 'World';
    • log(`Hello ${temp}`);
    • })();
    • << 'Hello World'
  • Use IIFE to enable Strict Mode. This would be useful for sharing code
    • (function() {
    • 'use strict';
    • // All your code would go inside this function
    • })();

Temporary Variables

  • You can’t remove a variable from scope once its been declared. You might have a variable that is only used temporarily, but you cant get rid of it after its use. Put the variable inside an IIFE which makes it available while the IIFE is invoked.
  • Example – variable temp is used within IIFE
    • let a = 1;
    • let b = 2;
    • (()=>{
    • const temp = a;
    • a = b;
    • b = temp;
    • })();
    • a;
    • << 2
    • b;
    • << 1
    • log(temp);
    • << Error: "temp is not defined"

Callbacks

  • Already covered, but they are functions passed to other functions as arguments and then invoked inside the function they are passed to
  • Instead of waiting for an event to occur, a callback can be created that’s invoked when the event happens. This way, code can run out of order (asynchronously).
    • Theses can be click and keypress DOM events
    • This helps ensure that waiting for these tasks to complete doesn’t hold up the execution of other parts of the program.
  • Promise: the future result of an asynchronous operation. They help simplify the process and avoid convoluted code from using multiple callbacks
    • Creating: use a contructor function.
      • const promise = new Promise( (resolve, reject) => {
      • // initialization code goes here
      • if (success) {
      • resolve(value);
      • } else {
      • reject(error);
      • }
      • });

Closures

  • Closure: a reference to a variable that was created inside the scope of another function, but is then kept alive and used in another part of the program.
    • Example: The outer() function only has access to the variable outside , which was declared in its scope. The inner() function, however, has access to the variable inside , declared in its scope, but also the variable outside , declared outside its scope, but from within the outer() function.
      • function outer() {
      • const outside = 'Outside!';
      • function inner() {
      • const inside = 'Inside!';
      • log(outside);
      • log(inside);
      • }
      • log(outside);
      • inner();
      • }

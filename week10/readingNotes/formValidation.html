Form Validation

  • Messages like:
    • "This field is required" (You can't leave this field blank).
    • "Please enter your phone number in the format xxx-xxxx" (A specific data format is required for it to be considered valid).
    • "Please enter a valid email address" (the data you entered is not in the right format).
    • "Your password needs to be between 8 and 30 characters long and contain one uppercase letter, one symbol, and a number." (A very specific data format is required for your data).
  • Client-side validation: done in the browser
  • Server-side validation: done on the server
  • Reasons why to validate:
    • We want to get the right data, in the right format
    • We want to protect our users data
    • We want to protect ourselves

Client-side Validation

  • Built-in form validation: HTML5 form features. Has better performance than JavaScript but not as customizable
  • JavaScript: completely customizable, but needs to be created

Built-in form validation

  • Use validation attributes on form elements:
    • Required
    • Minlength and maxlength: min and max length of strings
    • Min and max: min and max of numerical input types
    • Type: type of data like number, email, etc
    • Pattern
  • Example
    • Using required attribute: done in example below
    • Using pattern:
      • Expects a regular expression as its value (regex)
      • Regex examples
        • a — Matches one character that is a (not b, not aa, and so on).
        • abc — Matches a, followed by b, followed by c.
        • ab?c—Matches a, optionally followed by a single b, followed by c. ( ac or abc)
        • ab*c—Matches a, optionally followed by any number of bs, followed by c. ( ac , abc, abbbbbc, and so on).
        • a|b — Matches one character that is a or b.
        • abc|xyz — Matches exactly abc or exactly xyz (but not abcxyz or a or y, and so on).
      • Example done below

Validating forms using JavaScript

Constraint Validation API

  • Consists of a set of methods and properties available on these form element DOM interfaces:
    • HTMLButtonElement (represents a <button> element)
    • HTMLFieldSetElement (represents a <fieldset> element)
    • HTMLInputElement (represents an <input> element)
    • HTMLOutputElement (represents an <output> element)
    • HTMLSelectElement (represents a <select> element)
    • HTMLTextAreaElement (represents a <textarea> element)
  • Makes the following properties available on the elements above:
    • validationMessage: returns a message
    • validity: returns a ValiditiyState object containing properties of the validity state of the element.
